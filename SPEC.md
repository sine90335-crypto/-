# Spec：会读心的便签墙

## 背景

招聘任务的原始需求故意模糊：做一个“会读心的便签墙”。我先把它收敛为一个本地可运行、可交互、可验证的小产品。

核心目标不是做复杂的便签工具，而是展示 AI Native 的工作方式：用户输入内容后，系统能基于历史上下文给出有趣反馈，并且这个链路具备基本工程完整性。

本轮指定的长任务是：把“写下一张便签并得到 AI 回应”扩展成完整工作流，并把后端从单文件 demo 拆成可维护架构。它需要覆盖记录、整理、修改、搜索、筛选、置顶、删除、清空确认、AI 反馈保存、大画布缩放保存、热门便签精选，以及多组必须依赖 LLM 的能力。

## 用户故事

作为用户，我希望：

- 可以在页面上写一张便签并贴到墙上。
- 可以给便签选择此刻心情。
- 刷新页面后，之前的便签还在。
- 新增便签后，系统能结合已有便签给出一段“读心”反馈。
- 可以让 LLM 总结整面墙、找出便签关联、生成下一句提示。
- 可以搜索、筛选、置顶和编辑便签。
- 可以缩放便签墙，刷新后保留缩放比例；缩小时看到更多墙面，而不是把便签挤在一起。
- 当 AI 服务不可用时，便签仍然能保存，页面不会崩。
- 我可以删除不想保留的便签。

## 输入

- 用户输入的便签文本。
- 用户选择的心情标签。
- 已保存的历史便签列表。
- 可选的 LLM 配置：
  - `OPENAI_API_KEY`
  - `OPENAI_BASE_URL`
  - `OPENAI_MODEL`

## 输出

- 便签墙上的便签列表。
- 每张便签的心情、位置、颜色、置顶状态和时间。
- 便签墙画布状态，例如缩放比例、画布尺寸、热门精选阈值。
- 新增便签后的 AI 反馈：
  - `mood`：当前墙面的整体气质。
  - `thought`：对最新便签与历史内容关系的解读。
  - `prompt`：引导用户继续输入的下一步提示。
- API 返回明确的错误信息或 fallback 结果。

## 功能范围

必须实现：

- 前端页面可交互。
- 后端 API 可读写便签。
- 数据刷新不丢。
- 至少一次真实 LLM 调用能力。
- AI 功能必须走真实 LLM；未配置 API 时明确提示，不用本地规则冒充。
- README 写清启动方式、设计思路、AI 协作记录。
- 便签支持心情标签、搜索、筛选、编辑、置顶、删除。
- 便签墙支持大画布缩放，并通过后端持久化。
- 缩小到指定比例以下时，按热度展示精选便签，降低视觉拥挤。
- 清空所有便签前需要二次确认。
- 最近一次 AI 反馈需要随数据持久化。
- 后端需要拆成入口、路由、存储、业务服务和配置模块。

暂不实现：

- 登录、多用户、权限系统。
- 拖拽排版。
- 数据库迁移。
- 云端部署。
- 复杂心理分析或情绪诊断。

## 边界条件

- 空便签不能提交。
- 单张便签最多 240 字。
- LLM 未配置或调用失败时，新增便签和 AI 功能必须明确失败原因。
- 新增便签如果 LLM 失败，不保存伪 AI 结果，也不返回伪 AI 反馈。
- 删除便签失败时，前端要恢复原显示并提示用户。
- 编辑便签不能为空。
- 置顶失败时，前端要恢复原状态。
- 清空失败时，前端要恢复原便签列表。
- 缩放比例限制在 `0.6` 到 `1.5` 之间。
- 缩放保存失败时，前端要恢复原比例并提示。
- 热门精选不改变真实数据，只影响当前缩小视图的展示。
- 本地数据文件不存在时，后端要自动创建。

## 人工确认点

- 是否接受“读心”是一种轻量、有趣的上下文反馈，而不是心理评估。
- 是否接受 JSON 文件作为本地持久化方案。
- 是否接受 OpenAI-compatible 接口作为模型接入方式。
- 是否需要在提交前换成指定模型或指定 API 网关。

## 任务拆解

1. 项目初始化：创建 Vite + React + Express 项目结构。
2. 数据层：实现 `notes.json` 读写和自动初始化。
3. 后端架构：拆分 `config`、`store`、`routes`、`noteService`、`insightService`、`boardService`。
4. 后端 API：实现读取、新增、编辑、置顶、删除、清空、重新生成反馈、读取/保存画布状态。
5. LLM 接入：新增便签时调用 `chat/completions`，要求返回结构化 JSON，并保存最近反馈。
6. LLM 能力扩展：实现墙面总结、便签关联分析、下一句建议三个独立能力。
7. 前端交互：实现便签墙、新增表单、心情选择、搜索、筛选、编辑、置顶、删除、清空确认、大画布缩放控件、反馈面板和 LLM 工具区。
8. 异常处理：补空输入、LLM 未配置提示、编辑失败恢复、删除失败恢复、清空失败恢复、缩放保存失败恢复。
9. 视觉落地：参考 Apple 风格原型，做固定视口、玻璃卡片、网格画布、淡色便签。
10. 文档：补 README 和 spec。
11. 验证：安装依赖、构建、启动服务、请求 API、截图自查。

## 完成标准

- `npm install` 可以安装依赖。
- `npm run dev` 可以启动前后端开发服务。
- `npm run build` 可以完成生产构建。
- `npm start` 可以启动生产服务。
- 浏览器可以新增便签并看到反馈。
- 浏览器可以编辑、置顶、搜索、筛选和删除便签。
- 浏览器可以缩放便签墙，刷新后缩放比例仍存在。
- 缩小时便签仍保持分散，不应集中堆叠在画布中心。
- 便签很多时，缩小视图能自动精选热门便签展示。
- 清空便签需要二次点击确认。
- 刷新页面后便签仍存在。
- 刷新页面后最近一次反馈仍存在。
- 配置 API Key 后新增便签会触发 LLM。
- 不配置 API Key 时页面可打开，但新增便签和 AI 功能明确提示需要 LLM。
- 配置 API Key 后，墙面总结、关联分析、下一句建议均可调用。
- README 能让审阅者在本地复现项目。
